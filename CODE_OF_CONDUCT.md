# Code of Conduct

## Our Focus

TinyMUSH is a software development project focused exclusively on maintaining, improving, and supporting the TinyMUSH server. All project interactions must remain focused on this goal.

## Expected Behavior

All contributors, users, and participants are expected to:

- **Be Respectful**: Treat everyone with courtesy and professionalism, regardless of experience level or background
- **Be Constructive**: Provide helpful feedback, focus on solutions, and contribute positively to discussions
- **Be Welcoming**: Help newcomers get started, answer questions patiently, and foster an inclusive environment for learning
- **Stay On-Topic**: Keep all discussions focused on TinyMUSH development, usage, technical support, and related software topics

## Unacceptable Behavior

The following have no place in this project:

- **Politics**: Political discussions, opinions, or activism
- **Personal Opinions on Non-Technical Matters**: Social issues, religious views, or other non-technical topics
- **Off-Topic Discussion**: Subjects unrelated to TinyMUSH development and usage
- **Disrespectful Behavior**: Harassment, personal attacks, trolling, or inflammatory comments

## Scope

This Code of Conduct applies to all project spaces, including:

- GitHub repositories (issues, pull requests, discussions)
- Mailing lists and email communications
- Documentation and code comments
- Any other official TinyMUSH communication channels

## Appropriate Topics

Discussions should focus on:

- Code contributions and technical implementation
- Bug reports, fixes, and troubleshooting
- Feature requests and design proposals
- Documentation improvements
- Installation, configuration, and usage questions
- Module development and customization
- Performance optimization
- Security considerations
- Testing and quality assurance

## Non-Project Topics

For topics unrelated to TinyMUSH development and usage, please use appropriate external platforms. This project is not the venue for political, social, or other non-technical discussions.

## Enforcement

Project maintainers have the right and responsibility to:

- Remove, edit, or reject comments, commits, code, issues, and other contributions that violate this Code of Conduct
- Temporarily or permanently ban any contributor whose behavior is inappropriate, threatening, offensive, or harmful
- Close or lock discussions that go off-topic

Maintainers will focus enforcement on maintaining a productive, technical environment focused on TinyMUSH development.

## Reporting

If you observe or experience behavior that violates this Code of Conduct, please contact the project maintainers at:

**Email**: tinymush@googlegroups.com

Include:
- Description of the incident
- Where it occurred (link to issue, PR, etc.)
- Any relevant context

Reports will be reviewed and addressed as appropriate.

## Attribution

This Code of Conduct is focused specifically on maintaining a technical, development-oriented environment for the TinyMUSH project.
